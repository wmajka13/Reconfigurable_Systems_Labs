This project requires creating an automated test environment to verify the data serialization state machine designed in the previous task.

The testbench must read input data from a binary file, which should be manually populated with 16 random bytes or ASCII characters, and save the simulation results to an output binary file.

Additionally, a software reference model must be written in a language such as Python, C, C++, or MATLAB. This program will read the input file, serialize each character into a sequence of 0s and 1s representing its ASCII value, append the start and stop bits, and save the complete bitstream to a reference output file.

Finally, the output file generated by the Verilog testbench must be compared with the reference output file. If any discrepancies are detected, they must be documented, and the Verilog state machine module must be modified to correct the observed errors.
